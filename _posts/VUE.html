Mvvm & mvc

MVC

M：Model，数据
V：View，视图
C：Controller，控制器
特点：单向的


MVVM

M：Model，数据
V：View，视图
VM：ViewModel，视图模型
特点：双向的


MVVM是一个Model+View+ViewModel框架，Model是数据模型，ViewModel连接着Model和View，扮演着观察者的角色。
vue数据驱动
并不推荐直接操作DOM，操作的是数据

  • data更新view
    当数据发生变化时，ViewModel能够观察到数据的变化，通知到对应的view做视图更新；
    当数据改变时，会触发Object.defineProperty( )对属性设置的set函数，将更新view的方法放在这个set函数中。
  • view更新data
    当用户操作视图，ViewModel也能监听到变化，然后通知数据做改动，这就实现了view与model的双向绑定。
    因为view更新data其实可以通过事件监听即可，比如input标签监听 ‘input’ 事件就可以实现了。

优点：避免手动操作DOM更新

数组的响应

当data中的数据类型是数组时，Vue无法检测根据数组的索引和length修改数组的操作：

  • 当你利用索引直接更改一个项时，arr[index] = 值;
  • 当修改数组的长度时，arr.length = 值
    vm.ary[0] = 'a';

    vm.ary.length = 4;

    vm.$data.ary[2] = 33;//=>['a',2,33,undefined]

以上操作，vue都无法检测到，视图不会刷新，页面还是[ 1, 2 ]。

注意：数组中所有直接操作原数组的方法都可以刷新页面：
push / pop / unshift / shift / reverse / sort / splice
slice不可以

    vm.ary.reverse().reverse();

    // vm.ary.slice();//=>不可以

    // vm.ary = vm.ary.slice();//=>可以

对象的响应式

当data中的数据类型是对象时，vue会循环data对象中的属性，依次增加getter和setter。（数据劫持）

对象中的属性初始化了之后才会有set和get，有set和get的属性才是响应式的。

    vm.obj["sex"] = 0;


Vue.set()是Vue这个类作为对象的一个属性，使用Vue.set(target, key, value)为obj添加一个属性，代码如下：

    vm.$set(vm.obj, "age", 24);

注意：Vue.set()方法调用时，页面会全部渲染一遍，所以没有使用Vue.set()方法所做的更改也更新到了DOM，比如vm.arr.length = 4;

选项

computed

一个computed属性是依赖于一个或几个data中的数据“计算”得到的。
computed中的方法最后也会加到vm上，因此，不能和data、methods中的重名。

两部分组成：get和set语法：

computed: {

        checkAll: {

            get() {

                return 谁;

            },

            set(val) {

                谁 = val;

            }

        },

        sum() {

            return 值;

        }

},

这里需要注意的是：

computed计算“属性” 不是方法

  • get方法必须要有return。
  • 计算属性会被缓存，如果依赖的属性（归vue管理，响应式的）没有变化，就不会重新执行。
  • computed不支持异步，如果异步，get方法会return undefined;
  • 方法不会有缓存，computed会根据依赖(归vue管理的数据，可以响应式变化的)的属性进行缓存
  • 两部分组成有get和set(不能只写set) 一般情况下 通过js赋值影响其他人或者表单元素设置值的时侯会调用set方法

computed VS method

计算属性是基于它们的依赖进行缓存的。当它的依赖没有发生改变，会直接拿缓存的上次的执行结果，不会重新执行函数。。
而方法是没有缓存的，每当触发重新渲染时，方法总是不可避免的会被多次执行。

computed VS watch

computed不支持异步，依赖可以有多个，第一次进入页面默认执行。
watch支持异步，监控的对象只有一个，第一次进入页面默认不执行。

钩子函数之生命周期

每个Vue实例在初始化的过程中，到一定的阶段会调用的一些方法，这些方法叫做钩子函数。
声明周期：
beforeCreate created
beforeMount mounted
beforeUpdate updated
beforeDestroy destroyed

常用的钩子函数包括：

  1. beforeCreate
    • 触发：vue实例创建之后，并且初始化了内置的属性和方法（带$的）后执行
    • 注意：此时data、methods等都没有被初始化，无法直接访问。
      此方法一般用不到

  2. created

    • 触发：初始化注册器，实现根级别的响应式属性，data、methods等初始化完毕
    • 注意：DOM并未生成，el属性不存在（undefined）
      可以直接访问data和methods等，但无法获取DOM元素。
    • 适用于：Loading结束、Ajax请求、初始化操作

  3. beforeMount

    • 触发：
      • 保证有编译的元素：存在el属性或vm.$mount(el)被调用，满足一个条件即可
      • 保证有编译的模板：存在template属性或将外部html（app中的结构）作为模板
        如果存在template属性就会替换掉外部html，template属性优先级高。
        template只能有一个根元素，并且不能是文本节点（例如{{msg}}）。
    • 注意：
      • 模板挂载之前，完成了el属性的初始化，但并未挂载，仅仅是“占坑”。
        节点是虚拟的DOM节点。比如，data中有一个msg属性，占坑显示的是{{msg}}，并没有显示真实的DOM，绑定、指令等Vue的语法都还在上边。

很少用，没有什么实际意义

  1. mounted
    • 触发：模板挂载之后，el属性被新创建的vm.$el替换，真实的html渲染到了页面中，填坑完成。
    • 注意：真实DOM渲染完成，Vue相关指令、绑定都不存在了，可以操作DOM。

元素挂载后，如果是只读模式则所有Input只读。

  1. beforeUpdate 和 updated
    数据更新之前和更新后执行，虚拟DOM需要重新渲染和打补丁。
    注意：是页面依赖的数据发生了变化触发，非依赖数据变化不触发。一般用watch来代替这两个方法。

  2. beforeDestroy 和 destroyed
    vm.$destroy();被调用才会触发这两个函数。
    页面绑定的数据会解绑，但方法还可以调用
    销毁前可以清除定时器、清除事件绑定

vue生命周期详解

需要注意的一些细节问题：

  • 只有beforeUpdate 和 updated可能会被触发很多次，其它的钩子函数都是在初始化vue实例的那一瞬间只执行一次。
  • 父组件和子组件的生命周期先走谁的：
    • 父组件的beforeCreate、created、beforeMount

存在的问题：
动态的切换渲染组件，很消耗性能。当一个组件被切换显示时，会重新初始化一遍，created、mounted等钩子函数都会被执行一次。当该组件被切换隐藏时，组件会被销毁，beforeDestroy 和 destroyed会被触发。

解决方案：动态组件外面包裹一个keep-alive组件。

  • keep-alive会缓存不活动的组件实例，而不是销毁它们，避免重新渲染。
    缓存后不会再触发钩子函数。
  • keep-alive是一个抽象组件：它自身不会渲染为DOM 元素，也不会出现在父组件链中。

Girl.prototype.on = function (eventName,callback) {



    if(this._events[eventName]){ // 不是第一次

        this._events[eventName].push(callback); // {失恋:[cry,eat,shopping]}

    }else{

        this._events[eventName] = [callback] //{失恋:[cry]}

};

    // [].slice.call(arguments,1);

    // Array.from(arguments).slice(1);

    if(this._events[eventName]){

        this._events[eventName].forEach(cb=>cb(...args));

    }

};

实现单页开发的方式：

  • 浏览器自带的历史管理模式history
    history.go()
    history.pushState(”,”,’/b’) /b是一个虚拟的地址，页面不会跳转。但这个地址是不存在的，真实访问可能产生404错误，因此需要服务端支持。
  • 通过HASH记录跳转的路径
    好处：
    可以产生历史管理，前进后退页面不刷新
    更换hash值，页面不会跳转
    页面刷新会记录上次的hash状态
    hash方式不支持seo
    开发时使用hash模式，上线使用history方式
    (history.pushState())可能会导致404错误

开发时使用hash的方式 上线的话我们会使用history的方式

history模式和hash模式的区别

  • 初始化地址
    hash模式下，当我们创建了一个vue-router实例后，URL会多一个#/，现在的路径是从/后边开始算的，不再是原来的请求路径。例如，当我们在/后边输入home时，会切换到home组件
    history模式下，URL后面不会多#/
  • 切换后地址
    hash模式下，是在初始化的地址上加上切换的组件
    history模式下，直接丢掉原来初始化地址，在域后加上组件名，例如，切换到home组件为：http://localhost:63342/home
  • 渲染的区别
